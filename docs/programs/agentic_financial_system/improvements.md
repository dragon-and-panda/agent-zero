# Agentic Financial System - Improvement Backlog

This backlog tracks the highest-leverage improvements for turning the charter into an operating business system.

## P0 - Immediate

1. **Consent-first inbox intake spec**
   - Define required owner authorization, mailbox scope, export format, retention rules, and CRM target schema.
2. **CRM hygiene workflow**
   - Build the first extraction, dedupe, tagging, and export pipeline for owner-controlled data.
3. **Orange analysis playbook**
   - Document a repeatable Orange workflow for clustering, deduplication QA, and segment review.
4. **Listing concierge packaging**
   - Create offer tiers, sample deliverables, and a demo narrative using the existing listing service assets.
5. **Lead qualification rubric**
   - Add a lightweight qualification template for inbound opportunities so fulfillment stays focused on legal, high-intent work.

## P1 - Near-term

1. **Synthetic demo datasets**
   - Create safe mock inbox/contact datasets for demos, tests, and tutorials.
2. **Public-web research brief template**
   - Standardize sections, citations, pricing, and turnaround expectations.
3. **Metrics board**
   - Track acquisition source, close rate, average fulfillment time, gross margin, and repeat purchase rate.
4. **Content-to-offer funnel**
   - Link tutorials and story-driven videos to concrete call-to-action pages.
5. **Referral loop**
   - Formalize partnerships with agencies, operators, or resellers who can send compliant work.

## P2 - Later

1. **Productized CRM hygiene app**
   - Convert the service workflow into a repeatable software-assisted product.
2. **Template store**
   - Sell compliant prompt packs, intake templates, and SOP kits.
3. **Paper trading lab**
   - Only after recurring revenue stabilizes, create a separate research environment for simulated trading with strict risk review.
4. **Narrative brand system**
   - Turn the anthropomorphic mech-suit / robot-body story into a recognizable educational brand without misrepresenting results.
