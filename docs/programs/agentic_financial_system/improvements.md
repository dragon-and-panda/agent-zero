# Agentic Financial System Improvement Backlog

Rank improvements by expected ROI, legal clarity, implementation effort, and reuse
across multiple revenue lanes.

## Near-term improvements

1. Add an inbox-to-CRM extraction instrument that only operates on user-provided or
   explicitly authorized mail exports and emits structured JSON plus an audit log.
2. Add a pricing recommendation instrument for service packages using public market
   benchmarks.
3. Add a proposal generator that drafts ethical outreach offers for compliant lanes.
4. Add a metrics dashboard that tracks leads, conversion, revenue, delivery time,
   and refund rate.
5. Add a consent-verification checklist before any workflow touches customer data.

## Mid-term improvements

1. Add a portfolio router that allocates effort across multiple active revenue lanes.
2. Add telemetry for unit economics by lane: acquisition cost, fulfillment cost,
   gross margin, and payback period.
3. Add a lightweight experiment registry for pricing, offer framing, and onboarding.
4. Add policy-aware prompt fragments that auto-load when workflows mention email,
   scraping, outreach, or regulated financial topics.

## Deferred improvements

1. Add simulated capital allocation tooling for treasury management once service
   revenue is stable and reserves exist.
2. Add broker or payment-platform integrations only after sandbox validation and
   compliance review.
