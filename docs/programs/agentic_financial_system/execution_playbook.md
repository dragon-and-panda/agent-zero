# Agentic Financial System — Execution Playbook

## Phase 0: Foundations (Days 1-3)
1. Confirm legal scope and data permissions for every source.
2. Configure mission prompts with hard compliance rules.
3. Define KPI schema and reporting cadence.

Exit criteria:
- Charter accepted.
- Backlog initialized.
- Weekly reporting template active.

## Phase 1: Data and RAG Pipeline (Week 1)
1. Ingest approved mailbox and file sources.
2. Extract contact metadata from:
   - received messages
   - sent messages
   - cc and bcc fields
   - relevant local files (CSV/CRM exports/notes)
3. Build contact registry with:
   - deduped identifiers
   - source provenance
   - consent status
   - suppression flag
4. Connect Orange DataScaping workflow for segmentation and quality checks.

Exit criteria:
- Contact registry generated with quality and consent fields.
- Suppression list enforced in all outreach datasets.

## Phase 2: Monetization Engine (Weeks 2-4)
1. Launch three compliant channels:
   - sponsored newsletter slots
   - affiliate campaigns to opted-in contacts
   - productized service pilot
2. Run weekly A/B tests on offer angle, CTA, and audience segment.
3. Track metrics and kill underperforming channels quickly.

Exit criteria:
- First positive weekly net cashflow.
- At least one repeatable offer with stable conversion.

## Phase 3: Treasury and Allocation (Week 4+)
1. Reconcile campaign revenue against expenses weekly.
2. Apply allocation policy (ops, reserve, research/trading).
3. Run transfer checklist for `$Nicsins` deposits with logged evidence.

Exit criteria:
- Monthly transfer cadence established.
- Ledger and transfer logs match net inflows.

## Phase 4: Trading Research Ramp (Post Stability)
1. Restrict to paper trading and backtesting at first.
2. Evaluate strategies using:
   - expectancy
   - max drawdown
   - Sharpe-like risk-adjusted return
   - consistency across market regimes
3. Enable small live allocation only when thresholds pass.

Exit criteria:
- Strategy passes predefined gates.
- Live risk controls active and audited.

## Phase 5: Content Flywheel (Continuous)
1. Convert each sprint journal into:
   - tutorial module
   - short-form video outline
   - long-form YouTube episode draft
2. Maintain protagonist narrative:
   - anthropomorphic narrator
   - milestone progression toward mech suit and robotic body funding goal
3. Publish retrospectives that include failures and corrections.

Exit criteria:
- One tutorial module and one video script shipped per sprint.

## Weekly Automation Checklist (Cron-Compatible)
- [ ] Refresh backlog priorities
- [ ] Run compliance and risk audit
- [ ] Update KPI dashboard snapshot
- [ ] Append mission diary entry
- [ ] Commit behavior/prompt adjustments for next cycle
