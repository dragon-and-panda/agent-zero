# Agentic Financial System — Launch Plan

This plan converts the charter into an execution sequence with built-in redundancy and explicit stop conditions.

---

## 1. Launch Prerequisites
Do not launch a monetization lane until the following are true:
- the idea has been scored with `instruments/strategy/score.sh`,
- the idea is not blocked by `docs/policies/compliance_pack.md`,
- the delivery scope is written down,
- the customer consent path is defined when customer data is involved,
- and the success metric plus kill criteria are known.

---

## 2. Primary Revenue Lanes

### Lane A: Consent-Based Inbox-to-CRM Hygiene
**Offer shape**
- One-time cleanup package
- Optional recurring maintenance package

**Minimum deliverable**
- import customer-owned mailbox export or authorized account connection,
- extract business contacts only,
- deduplicate and label,
- export CRM-ready records with an audit summary.

**Why first**
- direct business value,
- clear transformation,
- high chance of repeatable automation,
- can generate case-study material for the content lane.

**Contingency**
- If mailbox access is too complex, fall back to CSV exports and manual upload flows first.
- If consent language is unclear, pause launch and finish the compliance spec before taking customers.

### Lane B: AI Listing Concierge
**Offer shape**
- listing optimization package,
- cross-posting add-on,
- response-assist add-on.

**Minimum deliverable**
- seller intake,
- image improvement,
- listing copy generation,
- publishing checklist or supported-platform syndication.

**Contingency**
- If channel automation is unstable, sell the service as assisted publishing instead of full autoposting.
- If platform rules change, keep image and copy services active while disabling the affected adapter.

### Lane C: Content and Course Flywheel
**Offer shape**
- tutorial series,
- lightweight paid playbook,
- later a full course or toolkit bundle.

**Minimum deliverable**
- publish the build journal,
- convert each completed workflow into a lesson,
- use the anthropomorphic narrator as a brand wrapper, not as a substitute for real proof.

**Contingency**
- If service sales are slow, increase output from this lane because every experiment can still become reusable content.

---

## 3. Execution Sequence

### Loop 1: Qualification
- Score the idea.
- Write the narrow service spec.
- Define inputs, outputs, and boundaries.
- Reject anything dependent on unconsented data extraction.

### Loop 2: Delivery Skeleton
- Build the smallest possible delivery workflow.
- Add a checklist, audit output, and failure notes.
- Prove the result can be reproduced.

### Loop 3: Content Capture
- Convert the delivery workflow into:
  - a journal entry,
  - a short script,
  - a tutorial or lesson,
  - and a reusable template or prompt pack.

### Loop 4: Redundancy
- Promote the strongest workflow into a second packaging format:
  - service -> template,
  - template -> course,
  - course -> lead magnet,
  - listing workflow -> managed service.

---

## 4. Loss Mitigation Rules
- Never rely on one revenue lane alone.
- Never rely on one provider, one platform, or one acquisition channel alone.
- Keep a reversible fallback for every automation:
  - API automation -> assisted/manual workflow,
  - live sync -> batch import/export,
  - full-service delivery -> template or audit-only offer.
- Prefer capped-scope packages over open-ended custom work.

---

## 5. Stop Conditions
Pause a lane immediately if:
- legality or consent becomes unclear,
- customer data requirements expand beyond the written scope,
- platform rules prohibit the planned automation,
- margins collapse under manual review load,
- or the only way to grow the lane is to use spam, personal-data resale, or deceptive claims.

---

## 6. Immediate Priority Order
1. Finalize the inbox-to-CRM hygiene service boundary and consent requirements.
2. Reuse the listing-service assets already present in the repository as the backup revenue lane.
3. Turn each program artifact and experiment into content inventory.
4. Expand only after at least two lanes can operate without depending on the same risk source.
