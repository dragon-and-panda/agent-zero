# Strategy Intake Queue

Use this file as the intake queue for new ventures, experiments, and monetization hypotheses. Every entry should be scored with `instruments/strategy/score.sh` before substantial execution begins.

## Scoring rubric

- Score each dimension from 0 to 10.
- Higher is better for: revenue, speed, automation, resilience, trust, capital efficiency, legal safety.
- Lower is better for: platform dependency.
- Any idea that depends on selling personal data, spam, or unclear data rights should be marked **REJECT** even if other scores look attractive.

## Review protocol

1. Add the opportunity to the table.
2. Run the scoring instrument.
3. Record the numeric output and recommendation.
4. Promote only GO or carefully bounded HOLD items into an active program backlog.
